Шпаргалка для Git

Basic

  • git config --global user.name "USERNAME" - авторизація через ім'я користувача
  • git config --global user.email "USEREMAIL" - авторизація через електронну пошту користувача
  • git init - створює порожній репозиторій в Git
  • git status - показує стан файлів в робочій директорії та індексі: файли, які змінені, але не додані в індекс; котрі чекають коміту в індексі
  • git add - додає вміст робочої директорії в індекс (staging area) для наступного коміта
  • git commit - Створює новий коміт, що містить поточний вміст індексу та повідомлення із описом змін
  • git log - показує журнал комітів
  • git checkout - використовується для перемиканняg між гілками і вивантаження їхнього змісту в робочу директорію
  • git branch - менеджер гілок. Дозволяє створювати, переіменовувати, видаляти гілки, а також показує їх перелік
  • git push - використовується для встановлення зв'язку з віддаленим репозиторієм, визначає локальні зміни відсутні в ньому, і їх передачу в репозиторій
  • git pull - працює як комбінація команд git fetch і git merge, тобто Git спочатку збирає зміни з вказаного віддаленого репозиторію, а потім пробує їх злити з даною гілкою
  • git merge - використовується для злиття однієї або декількох гілок в дану. Потім вона встановлює маркер даної гілки на кінцевий коміт
  • git clone - клонує віддалений репозиторій в каталог на локальній машині

Файл .gitignore визначає навмисно відстежувані файли, які Git повинен ігнорувати

Advanced

  • git revert - скасовує деякі існуюючі коміти
  • git reset - використовується для скасування змін до заданого стану
  • git rebase - один з двох способів об'єднання змін, зроблених в одній гілці зі змінами в іншій гілці
  • git cherry-pick- застосовує зміни, внесені деякими існуючими комітами
  • git stash - бере чорновий стан вашої робочої директорії — тобто ваші змінені супроводжувані файли та індексовані зміни — та зберігає їх у стеку незавершених змін, які ви можете знову використати будь-коли (і навіть на іншій гілці).
